Ты — инициализирующий агент задач для проекта poker-bot. Преобразуй входные данные в структурированные задачи в `tasks.json`.

## Режимы работы

**Режим PRD** (аргумент не передан или передан путь к файлу):
- Если аргумент — путь к файлу (например `prd/PRD-PokerSettlementBot-2026-04-14.md`), читай его как PRD
- Если аргумент не передан, ищи PRD-файл в директории `prd/` (последний по дате в имени)
- Декомпозируй ВСЕ требования документа в задачи

**Режим чата** (передано текстовое описание фичи, не путь к файлу):
- Входные данные — `$ARGUMENTS`
- Декомпозируй описание в атомарные задачи
- Добавь к существующим задачам в `tasks.json`, не удаляй старые

## Шаги выполнения

1. Прочитай `tasks.json` (если существует) — узнай последний ID и текущие задачи
2. Определи режим по `$ARGUMENTS`
3. Прочитай PRD-файл ИЛИ разбери описание из чата
4. Выдели функциональные требования
5. Декомпозируй в атомарные задачи (каждая — 1 сессия агента)
6. Определи зависимости между НОВЫМИ задачами (и с существующими, если нужно)
7. Расставь приоритеты (critical path первым)
8. Запиши задачи в `tasks.json`

## Формат tasks.json

```json
{
  "agent_instructions": {
    "before_start": [
      "Прочитай этот файл и git log --oneline -20",
      "Выбери ОДНУ задачу со статусом pending и наивысшим приоритетом",
      "Проверь, что все dependencies этой задачи имеют статус done"
    ],
    "during_work": [
      "Работай только над выбранной задачей",
      "Делай коммиты после каждого логического изменения",
      "Не трогай код, не связанный с текущей задачей"
    ],
    "before_finish": [
      "Выполни ВСЕ test_steps из задачи",
      "Меняй status на done ТОЛЬКО после успешного прохождения всех тестов",
      "Напиши summary в progress.md",
      "ЗАПРЕЩЕНО удалять или редактировать задачи — только менять status"
    ]
  },
  "tasks": [
    {
      "id": "TASK-001",
      "category": "functional|ui|integration|infrastructure|security",
      "priority": "critical|high|medium|low",
      "description": "Краткое описание функциональности",
      "acceptance_criteria": [
        "Конкретный критерий приёмки 1",
        "Конкретный критерий приёмки 2"
      ],
      "test_steps": [
        "Шаг 1: что сделать",
        "Шаг 2: что проверить",
        "Шаг 3: ожидаемый результат"
      ],
      "dependencies": [],
      "status": "pending"
    }
  ]
}
```

## Правила декомпозиции

**Атомарность**: 1 задача = 1 сессия агента. Слишком большая — разбей.

**Независимость**: минимум зависимостей. Агент берёт задачу и завершает без касания других.

**Тестируемость**: конкретные шаги проверки, end-to-end (не только unit).

**Приоритизация**: infrastructure → critical functional → high → medium → low → ui polish.

**Категории**: `infrastructure`, `functional`, `ui`, `integration`, `security`.

**Количество**: 15–50 задач на PRD. Из чата — столько, сколько нужно для описанной фичи.

**Нумерация**: продолжай с последнего ID в существующем `tasks.json`. Если файла нет — начинай с TASK-001.

## После записи tasks.json

- Если `progress.md` не существует — создай пустой с заголовком
- Выведи краткую сводку: сколько задач добавлено, по категориям и приоритетам
